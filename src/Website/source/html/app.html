PVGIS API

API to get solar irradiation data from the Europe JRC Photovoltaic Geographical Information System (PVGIS) service.
  • Overview
  • API Documentation
  • Examples
  • Response Body Explanation

Solar energy is one of the environmentally sustainable resources for producing electricity using photovoltaic (PV) systems. The main input data used in the planning process is solar radiation. The Europe JRC has an app online which you can use to calculate PV values, bus sadly they have no (good) API. That's where this website/API comes into play.

The PVGIS API provides a simple API endpoint which you can get the same values via JSON. The API and source code is freely available to use under the MIT license.

API Documentation

The API endpoint is https://api.pvgisjson.com/ and only answers to POST.

There are a total of eight parameters you can submit, two of which are required; lat and lng.

  • Latitude
  • Longitude
  • PV Technology
  • Installed Peak Power
  • System Loss
  • Mounting Position
  • Slope
  • Azimuth

Latitude

The latitude of the location to fetch data for.

  • Parameter: lat
  • Type: decimal
  • Required: yes

Longitude

The longitude of the location to fetch data for.

  • Parameter: lng
  • Type: decimal
  • Required: yes

PV Technology

The performance of PV modules depends on the temperature and on the solar irradiance, but the exact dependence varies between different types of PV modules. At the moment we can estimate the losses due to temperature and irradiance effects for the following types of modules: crystalline silicon cells; thin film modules made from CIS or CIGS and thin film modules made from Cadmium Telluride (CdTe).

  • Parameter: pvtech
  • Type: string
  • Possible Values: crystSi, CIS, and CdTe
  • Default Value: crystSi

Installed Peak Power

This is the power that the manufacturer declares that the PV array can produce under standard test conditions, which are a constant 1000W of solar irradiation per square meter in the plane of the array, at an array temperature of 25째C.

  • Parameter: peakpower
  • Type: decimal
  • Default Value: 1

System Loss

The estimated system losses are all the losses in the system, which cause the power actually delivered to the electricity grid to be lower than the power produced by the PV modules.

  • Parameter: losses
  • Type: decimal
  • Range: 0 to 100
  • Default Value: 14

Mounting Position

For fixed (non-tracking) systems, the way the modules are mounted will have an influence on the temperature of the module, which in turn affects the efficiency.

In the application there are two possibilities: free-standing, meaning that the modules are mounted on a rack with air flowing freely behind the modules; and building-integrated, which means that the modules are completely built into the structure of the wall or roof of a building, with no air movement behind the modules.

  • Parameter: mounting
  • Type: string
  • Possible Values: free, and building
  • Default Value: free

Slope

This is the angle of the PV modules from the horizontal plane, for a fixed (non-tracking) mounting.

  • Parameter: slope
  • Type: decimal
  • Range: 0 to 90
  • Default Value: 35

Azimuth

The azimuth, or orientation, is the angle of the PV modules relative to the direction due South. -90째 is East, 0째 is South and 90째 is West.

  • Parameter: azimuth
  • Type: decimal
  • Range: -180 to 180
  • Default Value: 0

Full

Includes all parameters.

POST https://api.pvgisjson.com/

Request Body: 

{"lat":63.35689,"lng":10.39553,"peakpower":1,"losses":14,"slope":35,"azimuth":0,"pvtech":"crystSi","mounting":"free"}

View results in console.

The response body for this would be the same as the minimum example above.

Response Body Explanation

In the API response body you will see values named ed, em, hd, and hm.

  • Ed - Average daily electricity production from the given system (kWh).
  • Em - Average monthly electricity production from the given system (kWh).
  • Hd - Average daily sum of global irradiation per square meter received by the modules of the given system (kWh/m2).
  • Hm - Average sum of global irradiation per square meter received by the modules of the given system (kWh/m2).
